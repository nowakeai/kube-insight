# Support

## Questions

Use GitHub Discussions for usage questions, investigation patterns, and design
questions once the public repository is available.

Until Discussions are enabled, open a normal issue only when the question has a
clear documentation or product gap.

## Bugs

Use the bug report issue template. Include:

- kube-insight version or commit
- command and sanitized output
- sanitized config snippets
- whether the issue affects ingestion, storage, query, API, MCP, or release

Do not attach generated SQLite databases, kubeconfig files, bearer tokens, or
unsanitized Kubernetes object payloads.

## Security

Do not use public issues for security reports. See [SECURITY.md](SECURITY.md).
